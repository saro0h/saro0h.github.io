Documentation around Symfony2 The documentation I read at least once.

Important il est important de lire la documentation provenant de symfony.com en anglais. En effet, la version anglaise est-celle qui est la plus à jour. Prenez soin de bien choisir la version de la documentation que vous consultez.
Nota Bene La documentation provient entièrement de la communauté travaillant bénévolement !

Pour contribuer à la version anglaise : symfony-doc
Pour contribuer à la version française : symfony-doc-fr

Cache HTTP English RFC 2616 - HTTP Cache Le must read ! Cette RFC permet de comprendre l'implémentation du cache HTTP du côté des navigateurs et des Reverses Proxy Cache (aka "RPC", comme Varnish, Squid…). Il vous faut la lire au moins une fois pour avoir une vision globale et "critique" de ce que vous aurez à faire avec les entêtes HTTP "Cache-Control", "ETag", "Last-Modified"… Par ailleurs, les modèles de Validation de cache et de d'Expiration y sont largement expliqué. Ceci dit, s'il le texte reste encore trop "théorique", pas d'inquiétude, les liens suivants vous aiderons à y voir plus clair !
Cache HTTP English Things Caches Do Ryan Tomayko vulgarise le cache HTTP en expliquant tout ce que l'on peut retrouver dans la RFC 2616.
Cache HTTP English/Français HTTP Cache - Book (Symfony2) Vous y retrouverez un rappel de ce que la RFC explique, puis une documentation orientée sur 4 axes :
  • Les types de caches : Publiques (navigateurs) et Privés (Gateway cache / RPC)
  • Une introduction au cache HTTP : rappel des headers importants et la configuration par défaut du cache dans une application développée avec Symfony2.
  • Les modèles de caching : Validation et Expiration et comment les mettre en place avec l'API riche du composant HttpFoundation de Symfony2.
  • Travailler avec les Edge Sides Includes : Vous devez avoir entendu parler d'ESI, une manière très intéressante de cacher des "morceaux" de page.
On y apprend tout particulièrement que Symfony2 intègre un reverse proxy cache entièrement implémenté en PHP !
Cache HTTP English/Français Intégrer Varnish dans sa stack - cookbook (Symfony2) On nous montre quoi implémenter côté Varnish (en Varnish Configuration Language) pour que les headers HTTP propres au cache soient correctement compris.
On y apprend tout particulièrement l'intérêt du header "X-FORWADED-*" et du coup, de la notion de "trusted proxies".
Cache HTTP English/Français Trusted proxies - cookbook (Symfony2) Trusted proxies - reference (Symfony2) Lorsqu'on place un RPC devant son application Symfony2, toutes les informations provenant du RPC sont placées dans les headers X-Forwarded-*. Symfony2 ne récupère pas naturellement toutes ces informations dans l'objet Request sans un minimum de configuration.
On y explique donc l'intérêt de la notion de "trusted proxies".
Cache HTTP English FOSHttpCache Bundle Le bundle FOSHttpCacheBundle offre de nombreuses fonctionnalités en plus de ce que le framework Symfony2 fait à la base. Ce bundle est tout indiqué pour une mise en place poussée du cache dans vos application Symfony2 (en particulier l'invalidation de cached qui peut être pain in the ass parfois ;) ).
